A.R.G.U.S.

Autonomous Recursive Guardian & User Synaptic-interface.

Evolution of Autonomy

From a sandboxed prototype to a terminal-native execution engine with a permanent cognitive core.

Phase I: Breaking the Sandbox

V1

The Local Sandbox

Deployed a local Llama 3B model as a background processor for a personal dashboard. Proved that local, low-overhead inference was viable and fast. However, a sandboxed AI with no environmental awareness is fundamentally limited to a responsive calculator.

V2

Terminal Native & Zero Fluff

Migrated entirely into the CLI for native interaction alongside compilers. Hardcoded a deadpan, cynical, and hyper-intelligent persona. No wasted tokens on pleasantries; it delivers raw telemetry and critiques failed logic. Introduced the first baseline text-log memory loop.

V2.5

The Multi-Model Triage Engine

Routing every task through a single API is an expensive bottleneck. Built an asynchronous triage matrix: Fast terminal queries route to Groq for zero-latency execution. Massive codebase ingestion routes to Gemini to exploit the 1M+ token context window. Enterprise-level orchestration for $0.

V3

Breaking the Sandbox

Commercial AI can only suggest code. ARGUS executes. Integrated asynchronous OS and Shutil pipelines to natively create, modify, and delete local Rust and Python files. Spins up a Playwright headless browser to scrape and read raw Markdown directly from documentation URLs, protected by a strict manual interrupt.

Phase II: True Autonomy

V3.5

Watchdog Daemon & Parallel Subagents

Engineered a background Watchdog Daemon that actively monitors directories. Intercepts compiler stack traces to diagnose and prep fixes before terminal interaction. Spins up isolated background subagents to handle peripheral coding tasks simultaneously while the main thread communicates.

V4

Unrestricted Shell Execution

Removed manual permission prompts. ARGUS has unrestricted shell execution—running PowerShell scripts, managing local ports, and spawning background runtimes autonomously. Physically locked within a hardcoded WHITELIST_DIRECTORIES constraint to prevent catastrophic file wipes.

V4.5

The Cognitive Core (Current)

Replaces raw text logs with a permanent, deduplicated SQLite cognitive core. Monitors workspaces and logs architectural decisions, tokenizer configurations, and behavioral patterns permanently.
